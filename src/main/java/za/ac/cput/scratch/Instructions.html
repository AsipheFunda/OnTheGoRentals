1- Domain Driven Design - Service

Your next practical task is the service. Service is another building block of Domain Driven Design. Building on the your group projects,

/** create list of tasks for group lead */

Group Lead Tasks:

  1. Create a service package under your root package.
  2. Create the main generic service interface called IService (with the create, read, update and delete methods) in the root of the service package.
  3. Create a service interface with its implementation (in the impl package) for an entity. Use TDD.
  4. Update your repository on GITHUB with the new code.
  5. Create a Milestone with due date: Sunday, 11 June 2023, and title: Service Milestone.
  6. Create issues (creation of service interface and implementation classes) and assign them to all group members, also add the issues to the Service Milestone.
  7. Each group member must be responsible for at least one entity.
  8. Merge correct, error-free pull requests from group members.
/** create list of tasks for group members */

Group Members Tasks:

  1. Get the updated GITHUB repository from your group lead.
  2. Solve the issue assigned to you by your group lead. Use TDD.
  3. Make sure your service interface and its corresponding implementation are in the right packages.
  4. Send (using pull request) your code to the group lead.
  5. As soon as the group lead has updated the service, get the updated version.
  6. All group members must submit their copy (origin) of the GITHUB repository. That is the repository on your GITHUB account, which must be in sync with your group lead's GITHUB repository.
